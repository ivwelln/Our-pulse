# Our pulse

`Our pulse` — iOS-приложение для быстрой диагностики состояния сети и косвенного определения режима «белых списков».

В некоторых регионах России могут включаться режимы «белых списков», при которых через мобильную сеть перестает открываться значительная часть привычных интернет-ресурсов. Главная проблема такой ситуации — внезапность.  Не известно заранее, когда именно ограничения начнут действовать и в какой момент доступ к нужным сайтам и сервисам окажется недоступен.

Это может создать вполне практическую проблему: человек может оказаться вне зоны Wi-Fi и именно в этот момент не получить доступ к важному ресурсу, который нужен ему срочно. Приложение помогает заранее заметить изменение состояния сети и хотя бы частично снизить эффект неожиданности. Оно показывает признаки включения или отключения режима «белых списков» и уведомляет пользователя о смене состояния, чтобы тот мог быстрее сориентироваться в ситуации.

## Как это работает

Приложение выполняет диагностику прямо на устройстве пользователя. Оно сравнивает доступность двух групп адресов: тех, которые в условиях ограниченного режима обычно продолжают открываться, и тех, которые в таком режиме обычно становятся недоступны. На основе этого сравнения приложение формирует текущее состояние сети, сохраняет короткий журнал наблюдений и, при необходимости, отправляет локальные уведомления.

Проверки выполняются локально не случайно. Режимы «белых списков» могут различаться в зависимости от региона, города и конкретного оператора связи, поэтому единый центральный сервер не смог бы отражать реальную картину для каждого пользователя. Только проверка с самого устройства позволяет увидеть то состояние сети, с которым пользователь сталкивается фактически здесь и сейчас.

Локальные уведомления также выбраны по практической причине. Во время действия «белых списков» push-уведомления могут не приходить, поскольку они зависят от серверов Apple, которые сами могут быть недоступны в режиме «белых списков». Поэтому приложение использует именно локальную логику определения состояния и локальные уведомления, не полагаясь на внешнюю инфраструктуру в тот момент, когда она может оказаться недоступной.

Приложение сравнивает доступность двух групп адресов:
- российских сервисов, которые в ограниченном режиме обычно остаются доступными
- зарубежных сервисов, которые при включении белых списков обычно перестают отвечать

На основе этих проверок приложение показывает текущее состояние сети, ведет короткий журнал и может отправлять локальные уведомления при смене состояния.

## Стек

- `Swift`
- `SwiftUI` для всего интерфейса
- `Observation` через `@Observable` для хранения состояния
- `Network` (`NWPathMonitor`) для системного мониторинга соединения
- `BackgroundTasks` (`BGTaskScheduler`, `BGAppRefreshTask`) для фоновых проверок
- `UserNotifications` для локальных уведомлений
- `CFNetwork` для определения VPN через scoped-интерфейсы системных proxy settings
- `URLSession` для HTTP-проверки контрольных адресов
- `Testing` для unit-тестов
- `XCTest / XCUI` для UI-тестов

## Основные возможности

- Показывает текущее состояние сети на главном экране
- Сохраняет результаты последних проверок в журнал
- Позволяет выбрать режим уведомлений
- Позволяет выбрать частоту проверок
- Предупреждает о включенном VPN, потому что он может искажать интерпретацию
- Работает с жизненным циклом приложения и поддерживает фоновые обновления

## Архитектура

- `ContentView` содержит `TabView` с тремя разделами: Главная, Журнал, Настройки
- `HomeTabView` показывает основной дашборд с текущим снимком состояния
- `JournalTabView` отображает сохраненные снимки
- `SettingsTabView` управляет режимом уведомлений и интервалом проверок
- `NetworkMonitor` содержит основную бизнес-логику, состояние, сохранение и отправку уведомлений
- `Our_pulseApp` передает `NetworkMonitor.shared` в SwiftUI environment
- `AppDelegate` регистрирует фоновую задачу и настраивает делегат уведомлений

`NetworkMonitor` хранит:
- последний снимок
- журнал снимков
- текущий тип соединения
- флаг VPN
- статус фонового обновления
- предпочтения по уведомлениям
- выбранный интервал проверок

## Адреса

По умолчанию приложение использует следующие адреса.

Российские / `allowed`:
- `https://ya.ru/robots.txt`
- `https://vk.ru/robots.txt`
- `https://www.gosuslugi.ru/robots.txt`

Зарубежные / `restricted`:
- `https://www.google.com/robots.txt`
- `https://www.cloudflare.com/robots.txt`
- `https://github.com/robots.txt`

Часть российских адресов может быть помечена как `isVPNRestricted`. Это означает, что при активном VPN такие адреса считаются менее надежными для интерпретации и сами по себе не дают достаточно оснований считать белые списки включенными.

## Как определяются белые списки

Основная идея — сравнение двух групп:
- российских адресов (`allowed`)
- зарубежных адресов (`restricted`)

В результате приложение вычисляет одно из пяти состояний.

### `whitelistOff`

- хотя бы один зарубежный адрес доступен

### `whitelistOn`

- хотя бы один основной российский адрес доступен
- ни один зарубежный адрес не доступен

### `offline`

Состояние означает: сеть недоступна или диагностика невозможна из-за полного отсутствия связности.

### `degraded`

Состояние означает: связность частичная или противоречивая.

Примеры:
- отвечают только VPN-чувствительные российские сервисы
- зарубежные сервисы отвечают, а российские нет
- наблюдается смешанная картина, которая не укладывается в ожидаемые паттерны

### `unknown`

Данных недостаточно для осмысленного вывода.

Сценарий ценарий:
- в конфигурации нет хотя бы одного российского и одного зарубежного адреса

## Уведомления

Приложение отправляет только локальные уведомления через `UNUserNotificationCenter`.

### Запрос разрешения

При первом подходящем сценарии приложение запрашивает разрешение на:
- alerts
- sound
- badge


## Важно!
- Определение белых списков носит эвристический характер и не является протокольным подтверждением политики оператора
- VPN может искажать как саму транспортную картину, так и итоговую интерпретацию
- Наблюдения по Wi‑Fi не всегда отражают поведение мобильной сети
- Время фоновых запусков контролируется iOS и не гарантируется точно
- Доступность конкретных сервисов может меняться по причинам, не связанным с белыми списками
